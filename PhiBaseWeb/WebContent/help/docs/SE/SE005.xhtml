Objetivo Mantenimiento de la estructura que representa a los usuarios del sistema. Aquí se deberán crear todos los usuarios que tendrán acceso al sistema.
Consideraciones 1. Para una correcta creación de usuarios, el sistema debe tener correctamente configurados los parámetros de envío de correo electrónicos que permitirán el envío de los datos iniciales de acceso al usuario creado.
Pantalla Principal
Parámetros Recibidos Ninguno.
Buscador 1. Por Usuario del Sistema.
Acciones 1. Adicionar Usuario del Sistema
2. Actualizar Usuario del Sistema
3. Eliminar Usuario del Sistema
4. Detalle Usuario del Sistema
5. Regenerar Contraseña
6. Desbloquear Usuario
Observaciones 1. La funcionalidad de "Regenerar Contraseña" solo se habilitará si el estado del Usuario es diferente a "Bloqueado por Intentos Fallidos".
2. La funcionalidad de "Desbloquear Usuario" solo se habilitará si el estado del Usuario es igual a "Bloqueado por Intentos Fallidos".
Adicionar Usuario del Sistema
Campo Descripción Tipo de Campo Consideraciones
E-Mail Usuario Dirección de E-Mail del Usuario. Alfanumérico-Email 1. Este campo es requerido.
2. El campo deberá seguir el formato de un E-Mail válido.
3. Este campo será el identificador único del usuario a crear.
Nombre del Usuario Nombre o descripción del Usuario del sistema. Alfanumérico 1. Este campo es requerido.
Perfil de Usuario Lista de los Perfiles de Usuario del Sistema. Alfanumérico-Lista 1. Este campo es requerido.
2. La lista se cargará con los Perfiles de usuario configurados en el sistema. Ver Mantenimiento Perfil de Usuario.
Observaciones 1. En el momento de crear el Usuario, se le asignará una contraseña por defecto que deberá ser cambiada en el primer Login del Usuario. El Usuario recibirá un E-Mail automático en donde se le indicarán sus credenciales de acceso al sistema.
Actualizar Usuario del Sistema
Campo Descripción Tipo de Campo Consideraciones
E-Mail Usuario Dirección de E-Mail del Usuario. OUTPUT 1. No es posible modificar su valor.
Nombre del Usuario Nombre o descripción del Usuario del sistema. Alfanumérico 1. Este campo es requerido.
Perfil de Usuario Lista de los Perfiles de Usuario del Sistema. Alfanumérico-Lista 1. Este campo es requerido.
2. La lista se cargará con los Perfiles de usuario configurados en el sistema. Ver Mantenimiento Perfil de Usuario.
Estado Usuario Estado del usuario dentro del sistema. Multivalor-Lista 1. Este campo es requerido.
2. Posibles valores:
   a. Activo.
   b. Inactivo.
3. Al estar el usuario Inactivo, no se permitirá su ingreso en el sistema.
Observaciones Ninguna.
Eliminar Usuario del Sistema
Observaciones 1. No será posible eliminar el Usuario si existen registros relacionados en las estructuras de "Login de Usuario", "Bloqueos de Usuario" o "Log de Queries Usuario".
Detalle Usuario del Sistema
Campo Descripción Tipo de Campo Consideraciones
E-Mail Usuario Dirección de E-Mail del Usuario. OUTPUT 1. No es posible modificar su valor.
Nombre del Usuario Nombre o descripción del Usuario del sistema. OUTPUT 1. No es posible modificar su valor.
Perfil de Usuario Perfil de Usuario asignado. OUTPUT 1. No es posible modificar su valor.
Estado Usuario Estado del usuario dentro del sistema. OUTPUT 1. No es posible modificar su valor.
2. Posibles valores:
   a. Activo.
   b. Inactivo.
   c. Inactivo Intentos Fallidos.
Intentos Fallidos Login Número de Intentos fallidos de ingreso en el sistema. OUTPUT 1. No es posible modificar su valor.
Contraseña Caducada Indicador que determina si el usuario deberá cambiar la contraseña en el siguiente ingreso. OUTPUT 1. No es posible modificar su valor.
Fecha Último Cambio Contraseña La fecha de último cambio de contraseña. OUTPUT 1. No es posible modificar su valor.
Observaciones Ninguna.
Regenerar Contraseña
Observaciones 1. Al ejecutar esta opción, se generará una nueva contraseña automáticamente para el usuario seleccionado y le será enviada por E-Mail. El usuario deberá cambiar la contraseña asignada en su siguiente inicio de sesión.
Desbloquear Usuario
Observaciones 1. Al ejecutar esta opción, el Estado del Usuario será "Activo nuevamente". Se actualizará el registro en la estructura de Bloqueos de usuario, indicando que el usuario ha sido desbloqueado manualmente por el usuario que se encuentra en sesión realizando la acción. El histórico de bloqueos se puede consultar en Bloqueos de Usuario
