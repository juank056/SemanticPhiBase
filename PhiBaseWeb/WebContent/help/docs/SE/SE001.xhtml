Objetivo Mantenimiento de la estructura que representa a los servicios del sistema.
En esta pantalla se podrán adicionar, modificar y eliminar los servicios del sistema Semantic PHI-BASE Web Interface.
Consideraciones La definición de los servicios del sistema, depende de las funcionalidades que ofrece la a aplicación.
Para incluir un nuevo servicio del sistema se debe contactar al desarrollador que debe indicar los parámetros necesarios.
Pantalla Principal
Parámetros Recibidos Ninguno.
Buscador 1. Por servicio del sistema
Acciones 1. Adicionar Servicio del Sistema
2. Actualizar Servicio del Sistema
3. Eliminar Servicio del Sistema
Observaciones Ninguna.
Adicionar Servicio del Sistema
Campo Descripción Tipo de Campo Consideraciones
Código Servicio del Sistema Código del Servicio del Sistema. Alfanumérico 1. Este campo es requerido.
2. Debe ser previamente definido por el desarrollador del sistema, ya que de esto depende el comportamiento y la autorización de los servicios.
Nombre Servicio Nombre del Servicio del Sistema. Alfanumérico 1. Este campo es requerido.
2. El nombre que se defina será el que se desplegará en el menú principal en caso de que el servicio sea desplegable en el menú.
Descripción Servicio Descripción del Servicio del Sistema. Alfanumérico 1. Este campo es requerido.
2. Esta descripción se desplegará como ayuda (ALT) sobre los servicios que sean desplegables en el menú principal de la aplicación.
URL Servicio URL del servicio. URL 1. Este campo es requerido.
2. La definición de la URL la dará el desarrollador del sistema. En caso de que sea un servicio que no es propio, se debe dar una URL completa de algún servicio de Internet.
Ícono Servicio Código y descripción del País. Alfanumérico 1. Este campo es requerido.
2. El ícono del servicio se deberá parametrizar de acuerdo a las imágenes contenidas dentro de la carpeta "images" del sistema.
Servicio Propio Indicador de si se trata de un servicio propio del sistema o si es un servicio externo. Booleano 1. Gracias a este campo, se podrán parametrizar, dentro del sistema, servicios que sean externos al mismo sistema. De esta manera se podrán incluir servicios que sean necesarios y que no sean propios.
Desplegable en Menú Indicador de servicio desplegable en el menú principal. Booleano 1. Este campo será utilizado para determinar si el servicio se podrá acceder desde el menú principal, o si se trata de un servicio que será accedido desde otro servicio.
Servicio Activo Indicador de servicio Activo. Booleano 1. Con este indicador se podrá activar o desactivar el servicio para permitir el ingreso de los usuarios dentro del mismo.
Observaciones Ninguna.
